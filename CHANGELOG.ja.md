# Change Log

## 0.3.11

- バッククォートを削除した後、メッセージの前後に残る余分な空白や改行も削除するように調整しました。

## 0.3.10

- 生成されたコミット- メッセージの前後に「`」が含まれている場合、それらを削除するようにしました。

## 0.3.9

- Codexスレッドで利用するモデル名を `gpt-5.1-codex-mini` に更新し、AI Modelの変更に追従しました。
- `@openai/codex-sdk` を 0.58.0 に引き上げ、最新SDKとの互換性を維持しました。

## 0.3.7

- Codex スレッドで使用するモデルを `gpt5-codex-mini` に固定した。

## 0.3.6

- VS Code の設定から Codex に渡す英語/日本語のイントロ文をカスタマイズできる項目を追加しました。
- 設定値を読み込んでプロンプトを構成し、空行を除去しつつ未設定のときは既定文にフォールバックするようにしました。

## 0.3.5

- 具体的なAIモデルの指定を廃止

## 0.3.4

- Codex の失敗や無効なメッセージ発生時に状況を明確に伝えるよう、コミットメッセージ生成のエラーハンドリングを強化しました。
- コミットメッセージを挿入するリポジトリをワークスペース状況に沿って選べるよう、リポジトリ選択ロジックを改善しました。
- Git コマンドにソフト上限と詳細なエラー出力を適用し、バッファあふれなどの失敗を防ぐようにしました。

## 0.3.3

- Marketplaceのカテゴリを「SCM Providers」に変更し、拡張機能を見つけやすくしました。

## 0.3.2

- ファイルサイズが以前として大きいため、Windowsのみを対象とした。

## 0.3.1

- ファイルサイズが大きくなりすぎたため、メジャーなプラットフォーム＆CPUの組み合わせのみとした。

## 0.3.0

- 拡張機能内で Git のステータスや差分を収集してから公式 `@openai/codex-sdk` を呼び出す構成に刷新し、外部の `codex_proxy` ヘルパーを不要にしました。
- これは、Windowsだけでなく、MacOSやLinuxでも動作するようにするための第一歩です。

## 0.2.5

## 0.0.18

- VSCode 拡張機能 "commit-message-gene-by-codex" の初回リリース。
- コマンド: "Commit message generation command execution" (`commit-message-gene-by-codex.runCodexCmd`).
- 小さな Windows ヘルパー (`codex_proxy.exe`) を介してローカルの `codex` CLI を呼び出し、Conventional Commits 形式のメッセージを生成。
- 生成メッセージをソース管理のコミット入力欄へ自動挿入し、ワークスペース直下の `.vscode-commit-message.txt` に保存。
- 診断用に "codex exec output" 出力チャンネルを提供。
